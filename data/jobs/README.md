# Boss 岗位数据分层

- `candidates.json`：已经发现、等待打开详情页核验的岗位线索。
- `verified.json`：公司、岗位、薪资、深圳归属和岗位详情直链均已核验的正式样本。
- `rejected.json`：重复、失效、非深圳、非月薪或无法核验的记录，并保留拒绝原因。

正式岗位库读取 `verified.json` 与仍在核验队列中的 `candidates.json`，但必须公开区分证据等级。分析快照对详情证据赋权 1.0、列表＋详情证据赋权 0.9、列表页观察赋权 0.55；`rejected.json` 永不参与分析。旧版页面中的岗位记录不能自动视为已核验样本。

岗位公开链接统一保存为当前公司岗位的 `job_detail` 详情页地址。同名岗位由不同公司发布时分别保留，不合并公司记录。

采集不绕过登录、验证码或平台限制，不保存账号、Cookie、联系方式等私人信息。
