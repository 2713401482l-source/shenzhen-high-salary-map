# V3 页面逻辑验收记录

验收日期：2026-07-13

## 数据边界

- 真实岗位库共 71 条，其中 7 条保存岗位详情要求，64 条为列表页观察。
- 演示数据共 12 个岗位方向，只保存在 `data/v3/demo/page-logic-demo.json`。
- 演示数据不含公司名和虚构来源链接，并声明 `excludedFromFormalStats: true`。
- 岗位证据库只读取真实岗位数据，薪资展示保留 Boss 原始区间。
- Boss 采集状态为暂停，V3 页面构建不会继续访问 Boss。
- 真实分析由 `npm run v3:analyze` 生成，页面不再自行临时拼接岗位族群和能力结论。
- 当前 2 个方向达到新兴信号门槛；最接近单岗对标的方向为 3/50 条详情证据。
- 4 条深圳政府官方资料单独保存为外部方向佐证，并标记 `excludedFromBossStatistics: true`，不进入岗位和能力统计。

## 页面逻辑

- 首页：用四步研究方法建立阅读顺序，并提供演示数据与真实证据切换。
- 发现机会：一次只展开一个岗位方向，避免同时堆叠多张榜单。
- 发现机会页增加真实市场侧写，薪资档、行业、区域、经验、学历五个维度共用一个切换面板，一次只展示一个维度。
- 市场侧写在 1440 × 1000 与 390 × 844 视口完成检查；手机页面 `scrollWidth` 为 375，无横向溢出。切换到“行业”后只展示前 10 个行业，并保留其余行业仍可在岗位库查看的提示。
- 发现机会页的官方资料只用于解释深圳产业投入方向，并明确声明不能代替岗位扩招证据。
- 能力组合：演示视图验证完整模块；真实视图只统计 7 条详情核验样本。
- 真实能力排行可以逐项选择，证据区只展示命中该能力的详情岗位；桌面和 390 像素手机视口均无横向溢出。
- 单岗对标：真实样本不足 50 条时显示空状态，不用演示数据补齐结论。
- 成长路径：演示视图验证机会排序和升档结构；真实视图仅展示现有样本分布。
- 企业多岗位信号保留公司、原始薪资和岗位直链，并明确标注为同一快照聚集，不称为持续扩招。
- 岗位证据库：仅真实数据，支持行业、区域、薪资和排序筛选，并直达当前公司岗位详情。
- 岗位证据库筛选后每页最多 12 条，切换筛选自动回到第一页；71 条真实岗位仍全部可达，不用无限长页面一次堆满。

## 视觉与交互

- 桌面端在 1280 像素宽度下检查全部核心页面，无页面级横向溢出。
- 手机端使用 390 × 844 的真实 iframe 视口检查首页、发现机会和岗位证据库。
- 手机菜单通过触屏坐标打开，`aria-expanded` 变为 `true`，背景滚动被锁定。
- 手机菜单打开后焦点进入关闭按钮，按 `Escape` 可关闭并把焦点交还菜单按钮。
- 手机菜单在展开动画结束时再次校正焦点，避免浏览器点击过程把焦点留在背景触发按钮。
- 手机筛选器改为单列，岗位卡片保持原始薪资、公司、区域、行业和详情链接。
- 390 × 844 视口复查岗位库：页面级 `scrollWidth` 为 375、无横向溢出；首屏结果为 12 条、共 6 页。翻到第 2 页后卡片内容更新，切换到 100K 筛选后自动回到第 1 页并显示全部 7 条匹配岗位。
- 机会排序把岗位方向、需求信号、薪资上限、样本数与能力门槛拆成独立列，并支持按综合参考、岗位数量和薪资上限排序。
- 能力门槛始终使用文字标签，不依赖颜色单独传达信息。
- 正式版准备度使用原生 `progress` 语义，屏幕阅读器可以读出当前值和目标值。
- 方法页按 30K、50K、100K 三档分别显示总样本与详情证据进度，避免把“有列表岗位”误读为“已经能做能力结论”。
- `prefers-reduced-motion` 下关闭 Shader 和滚动动画。
- 8 个页面构建产物都包含独立标题、Open Graph 描述、正式 canonical 地址和移动端主题色。
- 最终终验覆盖 `index / trends / skills / benchmark / growth / map / jobs / method` 8 个入口：390 × 844 与 1440 × 1000 两组视口均无页面级横向溢出、空白图表或控制台错误。
- 最终交互复查：结构演示可切换到真实证据；手机菜单展开后焦点进入关闭按钮，`Escape` 关闭后焦点回到菜单触发按钮并解除背景滚动锁定。
